# 비동기 작업

리포트 생성처럼 무겁고 오래 걸리는 작업을 백그라운드 워커가 처리한 실행 내역을 조회할 수 있는 페이지입니다.

<Screenshot src="/screenshots/operations/jobs.png" alt="비동기 작업 목록" />

## 주요 기능
### 작업 목록 테이블
가장 최근 작업 최대 50건을 표시합니다. 무거운 작업은 화면에서 즉시 실행되지 않고 백그라운드 워커로 넘겨지며, 이 페이지는 그 처리 결과를 **읽기 전용**으로 보여 줍니다.

| 컬럼 | 설명 |
|------|------|
| **Type** | 작업 종류 |
| **Status** | 처리 상태. 색상 배지로 표시됩니다 |
| **Runtime** | 작업이 실행된 위치 |
| **Error** | 실패 시 오류 메시지 |
| **Created** | 작업이 생성된 시각 (KST) |

### Status 배지
- **queued**: 대기열에 등록되어 처리를 기다리는 상태
- **running**: 워커가 처리 중인 상태
- **succeeded**: 성공적으로 완료된 상태
- **failed**: 처리 중 실패한 상태 (Error 컬럼에서 원인을 확인할 수 있습니다)
- **canceled**: 취소된 상태

### Runtime
- **lambda**: 짧게 끝나는 작업을 처리한 실행 환경
- **fargate**: 길거나 메모리를 많이 쓰는 작업을 처리한 실행 환경

## 사용 방법
1. 사이드바에서 **AI 운영 > 비동기 작업**을 클릭합니다
2. 테이블에서 최근 작업의 **Status**와 **Runtime** 배지를 확인합니다
3. 컬럼 헤더를 클릭해 원하는 기준으로 정렬합니다
4. **Refresh** 버튼을 눌러 최신 내역을 다시 불러옵니다. 마지막으로 갱신한 시각이 함께 표시됩니다

## 사용 팁
:::tip 실패 원인 확인
**Status**가 **failed**인 작업은 **Error** 컬럼에서 오류 메시지를 확인할 수 있습니다.
:::

:::info 읽기 전용 화면
이 페이지는 작업 실행 내역을 조회만 합니다. 작업은 다른 기능(예: 리포트 생성)에서 백그라운드로 등록되며, 처리 결과가 여기에 나타납니다.
:::

:::info 좁은 화면
화면이 좁아지면 테이블이 카드 형태로 전환되어 모바일에서도 보기 편합니다.
:::

# AWSops v1 → v2 기능 갭 종합 보고서

> 생성일 2026-06-10 · 방법: 12개 도메인 × (v1↔v2 코드 직접 매핑 → ADR/로드맵 적대적 검증 → 종합), 26 에이전트 워크플로우.

> 검증 방식: 도메인별 적대적 재검증(파일 증거 기반). v2 분류는 `terraform/v2/foundation/`, `web/`, `scripts/v2/`, `agent/lambda/` 실파일 grep/Read로 확정. partial/missing은 "코드 + 로드맵(CLAUDE.md P1~P4) + ADR 근거" 3중 확인으로 구분.

---

## 1) 한 줄 결론

**v1에 있으나 v2 코드에도 로드맵에도 없는 진짜 미구현(missing)은 총 36건이다.** 가장 사용자 가치가 큰 갭은 (a) **AI 종합진단+리포트(15섹션 DOCX/PPTX/PDF) 전체 도메인(11건)**, (b) **인프라/K8s 토폴로지 React-Flow 그래프(3건)**, (c) **Lambda EOL 런타임 경보·ECS 서비스/태스크·EBS 스냅샷 등 인벤토리 서브리소스(8건)**, (d) **Bedrock 토큰비용 모니터링 대시보드(3건)**, (e) **i18n 다국어 토글**, (f) **부서별 접근제어**다. 나머지 대부분은 partial(축소·UI비활성)·backlog(P3/P4 계획됨)·superseded(메커니즘 의도적 교체)로, "사라진 것"이 아니라 "다른 형태로 흡수되었거나 게이트 OFF" 상태다.

## 4) 🟡 부분 구현 (partial) — 동등물 존재하나 축소/비활성

### 4.1 제네릭 인벤토리로 흡수되며 "구조화 UI → JSON 덤프"로 격하된 상세 패널

- **WAF/CloudTrail/IAM role(trust policy)** 동일 패턴

### 4.2 KPI/차트 축소
- **EC2 요약**: 상태/타입 분포는 도넛으로 재현, but v1 "총 vCPU 합산(SUM core×threads)" KPI 동등물 없음. (대신 v1에 없던 평균CPU·시간당비용 카드 추가 — metrics route는 ec2만, 그 외 `{cards:[]}`)
- **EBS 인사이트**: 상태/타입 분포 1개 흡수, but 암호화율%/미암호화 'Action required'/유휴GB 낭비 KPI + 상태·암호화 별도 차트 손실(encrypted는 BAD_STATES 미포함이라 강조 안 됨)
- **ECR/S3/CloudFront**: 총개수+분포는 흡수, but scan-on-push 활성 카운트·HTTP-allowed 보안 KPI 등 derived 타일 손실
- **ElastiCache**: 클러스터만, Replication Group 별도 테이블·Valkey/Redis/Memcached 분리 KPI 없음
- **OpenSearch/MSK**: distKey가 engine_version/cluster_type일 뿐 암호화/브로커수/kafka_version KPI는 data JSONB에만(컬럼 미노출)

### 4.3 인벤토리 필터 축소

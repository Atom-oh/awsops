홈 · AI & AgentCore

라이브 AWS를 대화로 진단하는 AI 운영 레이어

Amazon Bedrock AgentCore 위에서 8개 섹션 게이트웨이가 실시간 AWS를 조회하고, 하이브리드 라우팅이 질문을 정확한 에이전트로 보냅니다. 모든 동작은 read-only로 고정됩니다.

AgentCore 런타임 개요

AWSops의 AI 레이어는 단일 AgentCore Runtime(Strands, agent/agent.py) 위에서 동작합니다. 이 런타임은 라이브 AWS를 조회하는 8개 섹션 게이트웨이를 호스팅하며, 게이트웨이 전체에 걸쳐 19개 Lambda 함수에 분산된 125개 MCP 도구를 노출합니다.

하이브리드 라우팅 + 합성

질문이 들어오면 정규식 fast-path가 명확한 케이스를 먼저 처리하고, 나머지는 Haiku 분류기가 라우팅합니다. 여기에 v2 프롬프트 캐싱이 더해집니다. 분류기는 1~3개의 라우트를 반환하며, 해당 게이트웨이들이 병렬로 호출된 뒤 답변이 합성됩니다.

라우팅 정확도 69.2% → 96.9% (+27.7pp) 프롬프트 캐시 적중 ~59% 응답 SSE 스트리밍
질문 → 정규식 fast-path / Haiku 분류기 (1~3 라우트) # 라우팅 → 게이트웨이 병렬 호출 → 답변 합성 → SSE 스트리밍 # 응답

AI 진단 티어

AI 진단은 두 가지 티어로 제공되며, 대시보드의 read-only collector를 그대로 재사용합니다.

01

Base 리포트

8개 섹션으로 구성된 기본 진단 리포트. 기본 모델은 Sonnet.

02

Deep 리포트

15개 섹션(8 base + 6 deep). deep 티어에서는 Opus 선택 가능(cost-gate).

멀티소스 자연어 쿼리

Monitoring 에이전트는 자연어 질문을 PromQL · LogQL · TraceQL · SQL로 변환합니다. 각 데이터소스의 스키마는 Aurora datasource_schemas 캐시에 저장되어 프롬프트에 주입됩니다.

커넥터(Prometheus Mimir Loki Tempo ClickHouse)는 SSRF 가드 뒤에서 read-only 쿼리 도구를 노출하며, 멀티소스 인시던트 상관분석을 지원합니다.

메트릭 → 로그 → 트레이스 → CloudTrail # 멀티소스 상관분석

read-only 거버넌스

◆

AWS 리소스 변경과 자율 동작은 영구 동결(do-not-enable)입니다. AWSops AI는 라이브 AWS를 조회하고 진단할 뿐, 리소스를 변경하지 않습니다.

외부 지식/커뮤니케이션 쓰기(티켓, 메시지 등)는 프로젝트 ADR에 따라 거버넌스 하에서만 허용됩니다.

  • DLP / redaction
  • 대상 allowlist
  • human gate
  • 기본 OFF

AWSops를 **새 아키텍처 위에 처음부터 다시 지은 v2.0.0**을 릴리스합니다. 단일 EC2 모놀리식을 **Terraform 기반 MSA**로 재구축했고, 그 위에서 AI 어시스턴트·AI 진단·토폴로지·EKS·비용 등 거의 모든 화면을 다시 만들었습니다. 사용자 입장에서는 전용 도메인·인앱 로그인·테마·모바일까지 경험 전반이 새로워졌습니다.

<!--truncate-->

:::tip 핵심 변화
**Terraform MSA 전환(ECS Fargate + Aurora)** · **비공개 엣지 + 인앱 로그인(RS256)** · **스트리밍 AI 어시스턴트(도메인 자동 라우팅·스레드 영속)** · **AI 진단 리포트(깊이 선택 · MD/DOCX/PDF)** · **요청 흐름 토폴로지 그래프** · **EKS 함대 + 인-클러스터 조회** · **3종 테마 + 모바일 반응형**
:::

## 새 아키텍처 (전면 재구축)

v1의 단일 EC2 호스트(빌드·운영·로컬 상태가 한곳에 묶여 있던 구조)를 풀어, 컴포넌트를 독립적으로 배포·확장할 수 있는 구조로 재설계했습니다.

| 영역 | 내용 |
|------|------|
| **IaC** | CDK → **Terraform** (S3 backend) |
| **컴퓨트** | 단일 EC2 → **ECS Fargate** (web/worker 분리, arm64) |
| **데이터** | 로컬 상태 → **Aurora Serverless v2 (PostgreSQL 17)** 영속 |
| **엣지** | 공개 ALB → **CloudFront VPC Origin → 내부 ALB → Fargate** (공개 ALB 없음, TLS end-to-end) |
| **인증** | 엣지 exp-only → **Cognito + Lambda@Edge RS256 JWKS 검증** |
| **장기 작업** | 인-프로세스 → **비동기 워커 티어** (SQS + Step Functions + Lambda/Fargate) |
| **경로** | `/awsops` 접두사 → **루트 경로(`/`)**, 전용 도메인 |

## 새로워진 경험

- **인앱 로그인** — 자체 `/login` 폼에서 이메일·비밀번호로 로그인합니다. 미인증 요청은 엣지에서 로그인 화면으로 안내합니다.
- **3종 테마** — **Cobalt(기본) · Teal · Dark** 테마 피커. 선택은 저장되며 차트·브랜드 마크까지 테마에 맞춰 바뀝니다.
- **모바일 반응형** — 상단 바 + 하단 5탭 + 슬라이드 드로어로 좁은 화면에서도 전체 기능을 사용할 수 있습니다.
- **명령 팔레트** — `Cmd-K`(또는 `Ctrl-K`)로 어느 페이지·리소스로든 바로 이동하고 테마를 전환합니다.

## AI — 더 빠르고, 더 똑똑하게

- **AI 어시스턴트** — 질문을 토큰 단위로 **스트리밍**하고 **마크다운(표·코드·목록)**으로 답합니다. 질문은 **도메인(네트워크·비용·보안 등)으로 자동 라우팅**되며, 대화는 **영속 저장**되어 드로어와 전체 화면이 같은 기록을 공유합니다.
- **AI 진단** — **Light / Mid / Deep** 깊이로 계정 운영 상태 종합 리포트를 생성하고, **MD · DOCX · PDF**로 내보냅니다. 의도 대비 실제(불변식) 점검과 이전 리포트 대비 변화 인사이트를 함께 제공합니다.

## 더 넓은 가시성

- **토폴로지** — Route53 → CloudFront → LB → Target Group → 타깃으로 이어지는 **요청 흐름 그래프**. 노드 클릭 시 포커스 모드 + 상세 패널(ARN 복사·AI 질문).
- **리소스 인벤토리** — EC2·Lambda·RDS·S3·VPC·IAM 등 **20여 종**을 정렬·검색·상세 조회.
- **EKS / Kubernetes** — 클러스터 **함대 요약**과 노드·파드·디플로이먼트 **인-클러스터 조회**(읽기 전용), OpenCost 설치 안내.
- **비용** — 서비스별 비용 분해·드릴다운, 월말 예측, 그리고 **Bedrock 모델 사용량**(토큰·지연·비용·캐시 절감).
- **데이터소스 탐색** — 연결된 관측성 데이터소스를 **네이티브 쿼리 언어(PromQL/LogQL/TraceQL/SQL)**로 읽기 전용 조회.

## 읽기 전용 운영 자세

AWSops는 운영 현황을 **관찰·분석·진단**하는 데 집중하며 **AWS 리소스를 변경하지 않습니다.** 설치·변경이 필요한 작업은 사용자가 직접 실행할 수 있도록 스크립트·안내를 제공합니다.

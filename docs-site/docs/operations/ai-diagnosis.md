# AI 진단

AWS 네이티브 데이터를 기반으로 계정 전반의 운영 상태를 분석한 종합 진단 리포트를 생성하고 열람할 수 있는 페이지입니다.

<Screenshot src="/screenshots/operations/ai-diagnosis.png" alt="AI 진단 리포트" />

## 주요 기능

### 진단 실행
- **진단 깊이(Tier)**: 분석 범위를 선택합니다.
  - **Light**: 핵심 항목만 빠르게 점검
  - **Mid**: 균형 잡힌 표준 진단 (기본값)
  - **Deep**: 약 15개 섹션을 폭넓게 분석. **Deep**에서만 사용할 모델(**Sonnet** / **Opus**)을 추가로 선택할 수 있으며, **Opus** 선택 시 비용 안내가 함께 표시됩니다.
- **진단 실행** 버튼을 누르면 리포트가 백그라운드에서 비동기로 생성됩니다.

### 진행 상태 표시
- 생성 중에는 **데이터 수집 → 섹션 분석 → 리포트 조립** 단계를 진행 바로 실시간 표시합니다.
- 완료되면 리포트가 자동으로 열립니다.

### 리포트 목록
왼쪽 사이드바에 최근 리포트가 나열됩니다:
- 자동 생성된 **제목**, **id**, **진단 깊이(Tier)**, **모델**, **상태(status)**, **생성 날짜**(KST)

### 리포트 본문
- 본문은 **목차(TOC)**와 함께 **마크다운**으로 렌더링됩니다.
- 상단 버튼으로 **MD / DOCX / PDF** 형식으로 내보내기할 수 있습니다.

### 인사이트 배지
- **의도 대비 실제 / 변화 인사이트** 배지 행이 불변식(invariant) 위반과 이전 리포트 대비 변화를 요약합니다.
- **의도 불변식 후보**(Intent) 패널에서 후보를 제안·수락·거부할 수 있습니다. (관리자 전용, 그 외 사용자에게는 읽기 전용)

### 제목 · 태그 · 삭제
- 리포트 작성자 또는 관리자는 **제목**을 인라인으로 수정하고, **태그**를 추가·삭제하며, 리포트를 **소프트 삭제**할 수 있습니다.

## 사용 방법

1. 사이드바에서 **AI 운영 > AI 진단**을 클릭합니다.
2. **진단 깊이**(**Light** / **Mid** / **Deep**)를 선택합니다.
3. **Deep**을 선택했다면 필요에 따라 모델(**Sonnet** / **Opus**)을 지정합니다.
4. **진단 실행** 버튼을 눌러 생성을 시작합니다.
5. 진행 바로 단계를 확인하고, 완료되면 자동으로 열리는 리포트를 확인합니다.
6. 본문 좌측 **목차**로 섹션을 이동하고, 상단에서 **MD / DOCX / PDF**로 내보냅니다.
7. 필요하면 **제목**·**태그**를 편집하거나 리포트를 삭제합니다. (권한이 있는 경우)

## 사용 팁

:::tip 중복 실행 방지
같은 조건(깊이·모델)으로 한 시간 이내에 다시 실행하면 새로 생성하지 않고 기존 리포트를 반환하며 중복 실행 안내가 표시됩니다.
:::

:::info 권한 안내
**제목 수정·태그·삭제**는 리포트 작성자 또는 관리자만 가능합니다. **의도 불변식 후보**의 제안·수락·거부는 관리자 전용이며, 다른 사용자에게는 읽기 전용으로 표시됩니다.
:::

:::info 읽기 전용 분석
AI 진단은 AWS 리소스를 관찰·분석할 뿐 변경하지 않습니다. 리포트는 분석 결과를 제공하는 용도입니다.
:::

## AI 분석 팁

리포트를 확인한 뒤, 플로팅 버튼이나 **AI 어시스턴트** 페이지에서 다음과 같이 후속 질문을 이어갈 수 있습니다:

- "이번 진단에서 가장 위험한 항목은 무엇인가요?"
- "이전 리포트 대비 어떤 변화가 있었나요?"
- "표시된 불변식 위반의 원인을 설명해 주세요."

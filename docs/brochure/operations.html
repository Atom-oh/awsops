한 번의 tfplan으로 안전하게 배포한다

AWSops는 Terraform 게이트와 멱등 프로비저너, ULID 마이그레이션으로 공유 인프라를 예측 가능하게 운영합니다. 기능은 플래그로 잠겨 있어 켜기 전까지 비용이 들지 않습니다.

프로비저닝 흐름

인프라 변경은 항상 terraform/v2/foundation/에서 진행하며, 저장된 tfplan을 적용하는 방식으로 게이트를 통과합니다. 공유 인프라에는 절대 -auto-approve를 쓰지 않습니다.

  1. configure — 대화형 TUI로 VPC·도메인·버킷·EKS를 고르면 terraform.tfvars와 backend.hcl이 생성됩니다.
  2. init — partial S3 backend 설정으로 초기화합니다.
  3. plan — 변경 사항을 tfplan 파일로 저장합니다.
  4. apply — 컨트롤러가 저장된 apply tfplan을 실행합니다. CloudFront·SG처럼 오래 걸리는 apply는 컨트롤러가 직접 돌립니다.
make configure # 대화형 TUI → terraform.tfvars + backend.hcl terraform -chdir=terraform/v2/foundation init -backend-config=backend.hcl terraform -chdir=terraform/v2/foundation plan -out tfplan # 컨트롤러가 apply tfplan
◆

공유 인프라에 -auto-approve 금지. 저장된 tfplan(apply tfplan)만이 자동 게이트를 통과합니다.

make 명령

일상 배포는 목적별 make 타깃으로 단순화됩니다. make deploy는 Terraform apply를 실행하지 않습니다.

make deploy # DB 마이그레이션 먼저 → arm64 빌드 → ECR push → ECS 롤링 → stable 대기 → /api/health 스모크 make agentcore # arm64 agent 이미지 + 멱등 프로비저너 (apply 후 실행) make workers # arm64 worker 이미지 push (workers_enabled=true로 apply 후)

마이그레이션 — ULID

스키마 변경은 terraform/v2/foundation/migrations/ 아래의 ULID 이름 파일로 관리됩니다. ULID 이름이라 충돌이 없고, advisory lock으로 동시 적용을 막으며, 버전 스탬프로 적용 여부를 추적합니다.

◆

make deploy가 먼저 적용합니다. 미적용 마이그레이션을 advisory-locked·version-stamped로 처리한 뒤에 web 이미지를 배포합니다.

컨테이너·엣지 운영 함정 (체크리스트)

재사용 가치가 큰 운영 교훈입니다. 배포 전 아래 항목을 확인하세요.

  • HOSTNAME=0.0.0.0은 런타임 env로. Next.js standalone 컨테이너는 task def environment에 명시해야 합니다. 이미지 ENV만으로는 부족 — ECS가 HOSTNAME을 ENI IP로 덮어써 ENI IP에만 바인딩되고 헬스체크가 UNHEALTHY가 됩니다.
  • Fargate 워커 Dockerfile은 CMD(ENTRYPOINT 금지). SFN containerOverrides.command는 CMD를 대체하지만 exec-form ENTRYPOINT엔 append되어 argv가 중복되고 argparse가 실패합니다.
  • health 경로 일치. 컨테이너와 타깃 그룹 health 경로가 앱(/api/health)과 같아야 합니다. 불일치 시 circuit breaker가 루프를 돕니다.
  • arm64 필수. web·agent·worker 이미지 모두 arm64로 빌드합니다.
  • SG description은 불변. 변경하면 SG replace가 발생해 연결된 ALB에서 hang됩니다. ingress 변경은 in-place로 하세요.
  • ECS secrets valueFrom은 실행 역할(execution role). task role이 아닙니다 — 아니면 ResourceInitializationError가 납니다.
  • SSM 예약어. "aws"로 시작하는 SSM 경로는 거부됩니다 → /ops/${project}/…를 사용합니다.

엣지 504→200 교훈

CloudFront→ALB는 TLS end-to-end여야 합니다. VPC Origin은 https-only + origin domain을 public FQDN으로 두어 SNI를 매칭하고, ALB는 HTTPS:443 + 리전 ACM, ALB SG는 CloudFront-VPCOrigins-Service-SG에서 443을 허용합니다. VPC Origin protocol은 in-place 변경이 불가하므로 create_before_destroy + -replace로 교체합니다.

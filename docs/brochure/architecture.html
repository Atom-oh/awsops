홈 · 아키텍처

읽기 전용 운영 대시보드를 위한 비공개 엣지 + AI 아키텍처

AWSops는 Terraform으로 정의된 단일 파운데이션 위에 비공개 엣지, Aurora 영속 상태, AgentCore 기반 AI 진단을 결합합니다. AWS 리소스 변경과 자율 실행은 영구 동결된 읽기 전용 설계입니다.

개요 및 포지셔닝

AWSops는 실시간 AWS·Kubernetes 운영을 한곳에서 관찰하고 AI로 진단하는 읽기 전용 운영 대시보드입니다. 전체 인프라는 terraform/v2/foundation 단일 루트에서 Terraform으로 관리되며, 상태는 S3 partial backend(awsops-v2-tfstate 버킷, use_lockfile, DynamoDB 미사용)에 저장됩니다.

비공개 엣지 — 공개 ALB 없음

외부 진입점은 CloudFront 하나이며, 백엔드 ALB는 외부에 노출되지 않습니다. CloudFront에서 Fargate까지 TLS가 종단 간으로 유지됩니다.

CloudFront TLS VPC Origin https-only:443 내부 ALB HTTPS:443 · 리전 ACM Fargate awsops-v2-web:3000
  • ALB 보안그룹은 CloudFront 관리형 SG CloudFront-VPCOrigins-Service-SG에서 들어오는 443만 허용합니다.
  • VPC Origin은 https-only로 설정하고, 오리진 도메인은 공개 FQDN을 사용해 SNI를 일치시켜 종단 간 TLS를 보장합니다.
  • 공개 ALB가 존재하지 않으며, ALB는 HTTPS:443 + 리전 ACM 인증서로 종단합니다.

AI — AgentCore + SSM

AI 진단은 Amazon Bedrock 모델과 AgentCore Runtime(Strands) 위에서 동작합니다.

모델 Sonnet 5 모델 Opus 4.8 모델 Haiku 4.5
  • AgentCore Runtime(Strands) + 8개 섹션 게이트웨이 + Memory + Code Interpreter로 구성됩니다.
  • 설정의 source of truth는 SSM /ops/awsops-v2/agentcore/{runtime_arn,interpreter_id,memory_id}이며, BFF가 런타임에 읽어 들입니다.

비동기 워커 티어

OOM 위험이 있거나 오래 걸리는 작업은 워커 티어로 안전하게 분리됩니다.

  1. 웹 POST /api/jobs → worker_jobs에 queued 행 기록 + SQS 적재
  2. Event Source Mapping(킬 스위치) → dispatcher Lambda(job_id 기준 멱등)
  3. Step Functions Standard, $.runtime 기준 Choice 분기
  4. 짧은 작업은 RunLambda, 길거나 OOM 위험 작업은 Fargate의 ecs:runTask.sync
  5. 워커가 직접 running / succeeded를 기록
  6. Catch 시 status_updater Lambda가 failed로 표시 (SFN은 VPC Aurora에 쓸 수 없음)
  7. reaper(EventBridge, 5분마다)가 stale 작업을 정합화

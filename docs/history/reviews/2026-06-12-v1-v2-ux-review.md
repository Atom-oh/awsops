## 1. 총평

사용자 직감 **"CSS 빼고는 v1이 마음에 드는 게 많다"는 데이터로 확인됨** — 차이의 본질은 *정보 밀도와 운영 신호*다. v1은 못생겼지만 "지금 뭘 봐야 하는가"를 말해줬고, v2는 아름답지만 평평한 카운트를 보여준다. 단, v2가 구조적으로 우월한 영역(AI 진입점, 부분-장애 격리, 일관성, 검색/정렬)은 v1로 되돌릴 이유가 없다 — **개선 방향은 "v2의 그릇에 v1의 신호를 다시 붓기"다.**

### V1이 여전히 이기는 곳
| 영역 | v1 | v2 현재 |
|---|---|---|
| 홈 운영 신호 | KPI 20개+파생 서브라인 + **Active Warnings 패널**(공개 S3·no-MFA IAM·열린 SG·알람, 딥링크) | 플랫 카운트 15개, 경고 표면 **0** |

| 인벤토리 깊이 | 타입별 맞춤 페이지(RDS 라이브 CloudWatch 그리드+스파크라인, EBS 스냅샷 탭+FinOps KPI, S3 보안 TreeMap, EC2 컬럼별 필터) | 단일 제네릭 donut+KPI+table (메트릭은 EC2 전용 2카드) |
| 챗 투명성 | 메시지별 토큰/비용/지연 + 도구 사용 표시 | 서버에 데이터 있으나 미표시; 답변 중 "얼어붙은" 느낌(중간 status 없음) |
| 비용 깊이 | 컨테이너 비용 페이지, 서비스 드릴다운 | MTD+추이만; ECS 비용 페이지 부재 |
| K8s 노드 상세 | ENI/IP 슬롯 + ENI별 트래픽, 노드별 pod 목록 | (어제 v1-parity 재구축으로 대부분 해소) 노드 클릭=플랫 키/값 패널, ENI 뷰 부재 |

## English Summary

Six-axis parallel code-reading comparison (7-agent workflow) + completeness critic + 3-AI panel prioritization challenge. The user's instinct — "v1 was better except the CSS" — is confirmed and localized: the delta is **operational signal density** (v1's warnings panel, derived KPI sublines, live metric grids, refresh affordances) vs v2's beautiful-but-flat counts. v2's structural wins (AI entry, partial-failure isolation, consistency, Cmd-K) are keepers; the direction is *pour v1's signal back into v2's vessel*. Panel-unanimous top-5: **C02 sign-out/identity trust fix, C01 universal refresh+staleness, C05 home warnings card, C30 cost service drill-down, C06 KPI hint sublines** (C09 metric registry as #6). Panel net-new finds: global resource search (C41, kiro), CSV export (C38), saved views (C42), zero-state onboarding (C40), session-expiry lock screen (C43). 43 candidates total, all read-only-posture compatible; mutation-adjacent v1 features (account selector, alert-settings writes, OpenCost install) deliberately not pursued per the 2026-06-11 ADR reversal.

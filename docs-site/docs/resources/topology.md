# 토폴로지

요청이 흐르는 경로(**Route53 → CloudFront → Load Balancer → Target Group → 타깃**)를 인터랙티브 그래프로 탐색할 수 있는 페이지입니다.

<Screenshot src="/screenshots/resources/topology.png" alt="요청 흐름 그래프" />

## 주요 기능
### 요청 흐름 그래프
- **Route53 → CloudFront → Load Balancer → Target Group → 타깃**으로 이어지는 트래픽 경로를 노드와 엣지로 시각화합니다.
- 노드는 종류별 색상과 아이콘으로 구분되며, 타깃 노드는 **healthy / unhealthy / draining** 등 health 상태에 따라 색이 바뀝니다.
- 그래프 상단에 현재 **노드 수**와 **엣지 수**, 그리고 인벤토리 동기화 시각이 표시됩니다.
- 화면 우하단의 **MiniMap**과 좌하단 **Controls**로 자유롭게 이동(pan)/확대(zoom)할 수 있습니다.

### 진입점 필터
- 상단의 **CloudFront** 선택 상자로 특정 배포를 고르면, 그 진입점에서 시작하는 경로만 남도록 그래프를 좁힐 수 있습니다.
- **LB** 선택 상자로 특정 Load Balancer 기준으로도 동일하게 좁힐 수 있습니다.
- 두 선택 상자 모두 **전체**로 두면 그래프 전체가 표시됩니다.

### 리소스 검색
- 상단 검색창에 리소스 이름의 일부를 입력하면 자동완성 목록이 나타납니다.
- 목록에서 항목을 고르면 해당 노드로 곧바로 포커스됩니다. **Enter**는 첫 번째 일치 항목을 선택합니다.

### 포커스 모드 + 상세 패널
- 노드를 클릭하면 **포커스 모드**로 전환되어, 연결된 상위/하위 경로만 남기고 화면 중앙에 다시 정렬됩니다.
- 동시에 오른쪽 **상세 패널**이 열려 해당 리소스의 필드를 보여줍니다. **VPC / subnet / security group ID**는 사람이 읽기 쉬운 이름으로 함께 표시됩니다.
- 패널에서 **ARN 복사** 버튼으로 리소스 식별자를 복사하고, **AI에 질문** 버튼으로 AI 어시스턴트에 바로 질문할 수 있습니다.
- 리소스 종류에 맞는 **추천 질문 칩**이 함께 제공되며, 네트워크 배치가 있는 리소스에는 **관계 그래프** 링크가 표시됩니다.
- 빈 공간을 클릭하면 선택이 해제되고 전체 그래프로 돌아옵니다.

<Screenshot src="/screenshots/resources/topology-detail.png" alt="노드 포커스 모드 + 상세 패널" />

## 사용 방법
1. 사이드바에서 **리소스 > 토폴로지**를 클릭합니다.
2. 그래프가 그려지면 **MiniMap**과 **Controls**로 원하는 영역을 확대해 살펴봅니다.
3. 특정 진입점만 보려면 상단 **CloudFront** 또는 **LB** 선택 상자에서 대상을 고릅니다.
4. 찾는 리소스가 있으면 검색창에 이름 일부를 입력하고 자동완성 목록에서 선택합니다.
5. 노드를 클릭해 **포커스 모드**로 들어가고, 오른쪽 **상세 패널**에서 필드를 확인합니다.
6. 필요하면 **ARN 복사**, 추천 질문 칩, **AI에 질문**, **관계 그래프** 링크를 활용합니다.
7. 빈 공간을 클릭해 선택을 해제하고 전체 그래프로 돌아옵니다.

## 사용 팁
:::tip 진입점부터 따라가기
한 서비스의 경로 전체를 보고 싶다면 **CloudFront** 또는 **LB** 선택 상자로 진입점을 고른 뒤, 끝단 타깃까지 흐름을 따라가 보세요. 타깃 노드 색으로 health 상태를 한눈에 파악할 수 있습니다.
:::

:::info 표시 시각
그래프 상단의 인벤토리 동기화 시각과 상세 정보의 시각은 모두 한국 표준시(KST, Asia/Seoul) 기준입니다.
:::

## AI 분석 팁
상세 패널의 추천 질문 칩이나 **AI에 질문** 버튼을 사용하면, 선택한 리소스의 맥락이 채워진 채로 AI 어시스턴트가 열립니다. 예시 질문:
- 이 CloudFront 배포가 오리진과 TLS로 통신하나요?
- 이 Load Balancer의 리스너/타깃 health 상태가 왜 이런가요?
- 이 Target Group의 unhealthy 타깃 원인을 진단해 주세요.
- 이 IP가 속한 인스턴스/ENI와 보안그룹을 확인해 주세요.

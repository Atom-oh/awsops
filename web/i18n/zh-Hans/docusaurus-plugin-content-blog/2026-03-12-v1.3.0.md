**85 个提交** · **18 个仪表板卡片** · **多路由（1~3 个 Gateway 并行）** · **SSE 实时流式传输** · **3 个新页面**
:::

## 多路由 AI 架构

分析复合问题后**同时**调用最多 3 个 Gateway，并由 Bedrock 合成响应。

<MultiRouteSequence />

| 项目 | Before (v1.2.0) | After (v1.3.0) |
|------|-----------------|-----------------|
| 问题处理 | 仅调用单个 Gateway | 并行调用 1~3 个 Gateway |
| 复合问题 | 仅使用第一个匹配的路由 | 并行处理所有相关路由 |
| 响应合成 | 无 | Bedrock `synthesizeResponses()` |
| 流式传输 | 无 | SSE `chunk` 事件实时发送 |

:::info 多路由工作原理
`src/app/api/ai/route.ts` 中的意图分类器分析问题后返回 `routes: string[]` 数组。
当数组长度大于等于 2 时，通过 `Promise.allSettled()` 并行调用，并使用 `synthesizeResponses()` 合成成功的响应。
:::

## 仪表板全面重新设计

将 18 个卡片以 6x3 布局排列，并与侧边栏菜单实现 1:1 映射。

<DashboardCardsLayout />

- **卡片底部 sub-metrics**：以 `18 running · 4 stopped` 形式显示核心数据
- **CIS Compliance Pass Rate**：整体基准测试通过率仪表
- **Monthly Cost**：本月费用 + MoM 变化率
- **Resource Distribution + K8s Pod Status**：使用 Recharts 图表可视化

## SSE 流式 AI 响应

通过实时流式传输 AI 响应改善了 UX。

<SSEEventsFlow />

- **`status` 事件**：显示分析中 → Gateway 连接 → 工具调用等阶段
- **`chunk` 事件**：实时发送响应文本
- **`done` 事件**：最终结果 + 使用的工具 + 响应时间 + Token 数
- 新增响应时间显示、剪贴板复制、相关推荐问题功能

## 新页面（3 个）

| 页面 | 路径 | 主要功能 |
|--------|------|-----------|
| CloudFront | `/cloudfront-cdn` | Distribution 列表、Origin、Behavior、缓存策略 |
| WAF | `/waf` | WebACL 规则、IP Set、拦截日志 |
| ECR | `/ecr` | 仓库、镜像标签、漏洞扫描结果 |

## VPC & Network 增强

- **VPC Resource Map**：AWS 控制台风格的 4 列布局
  ```
  VPC → Subnets → Route Tables → Connections (IGW/NAT/TGW/VPN)
  ```
- **Route Table 选项卡**：子网关联、路由规则表格
- **TGW Route Tables**：Transit Gateway 路由表详情
- **Resource Topology**：基于 React Flow 的 Graph View / Map View 切换

## EKS & Cost 增强

**EKS 改进**：
- K8s Overview 节点卡片：CPU/Memory 进度条
- 节点详情：ENI 卡片 + per-ENI 流量 + Pods 表格
- EKS Explorer：Status/Node 过滤器、分页

**Cost Explorer 改进**：
- 时间段/服务过滤器、Projected 月末费用、MoM Change
- Monthly Trend 折线图、Cost by Service 饼图

:::warning Steampipe Cost 查询注意事项
Cost Explorer 数据仅在 **MSP（Managed Service Provider）Payer 账户**中可用。
请先通过 `checkCostAvailability()` 进行确认，如不可用则使用 `cost-snapshot.ts` 的回退数据。
:::

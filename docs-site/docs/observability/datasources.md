# 데이터소스 탐색

연결된 관측성 데이터소스를 각각의 네이티브 쿼리 언어로 읽기 전용 조회할 수 있는 페이지입니다.

<Screenshot src="/screenshots/observability/datasources.png" alt="연동 허브 — 데이터소스 탭" />

## 주요 기능

### 지원 데이터소스와 쿼리 언어
연결된 데이터소스를 그 종류에 맞는 네이티브 언어로 직접 조회합니다. 모든 조회는 **읽기 전용**입니다.

| 데이터소스 | 쿼리 언어 | 예시 |
|-----------|----------|------|
| **Prometheus** | **PromQL** | `rate(node_cpu_seconds_total[5m])` |
| **Mimir** | **PromQL** | `up` |
| **Loki** | **LogQL** | `{job="varlogs"} \|= "error"` |
| **Tempo** | **TraceQL** | `{ duration > 500ms }` |
| **ClickHouse** | **SQL** (읽기 전용) | `SELECT count() FROM system.tables` |

### 데이터소스 선택
- 드롭다운에서 조회할 데이터소스를 고릅니다. 각 항목에는 **slug**와 **종류(kind)**가 함께 표시됩니다.
- 스키마가 미리 수집된 데이터소스에는 **스키마 캐시됨** 표시가 붙습니다. 이 캐시는 **AI로 생성** 기능이 실제 이름을 사용하도록 돕습니다.

### 시간 범위(range) 토글
- **Prometheus**, **Mimir**, **Loki**처럼 범위 조회를 지원하는 종류를 선택하면 **시간 범위 (range)** 체크박스가 나타납니다.
- 끄면 단일 시점(instant) 조회, 켜면 시계열(range) 조회로 동작합니다.

### 결과 표시
- 시계열(matrix/range) 결과는 **시계열 영역 차트**로 그려집니다.
- 단일 시점·로그·트레이스·SQL 결과는 정렬 가능한 **테이블**로 표시됩니다.
- 결과가 상한에 도달하면 **결과가 잘렸습니다** 경고가, 데이터가 없으면 **결과 없음** 안내가 명확히 표시됩니다.

### AI로 쿼리 생성
- 원하는 내용을 자연어로 설명하고 **AI로 생성**을 누르면, 캐시된 스키마를 참고해 해당 언어의 쿼리로 변환되어 입력창에 채워집니다.
- 생성된 쿼리는 **자동으로 실행되지 않습니다.** 검토 후 직접 **실행**을 눌러야 조회됩니다.

## 사용 방법
1. 사이드바에서 **연동**을 클릭한 뒤 **데이터소스** 탭에서 조회할 데이터소스의 **Explore**를 엽니다
2. 상단 드롭다운에서 조회할 **데이터소스**를 선택합니다
3. (선택) 범위 조회가 가능한 데이터소스라면 **시간 범위 (range)** 를 켭니다
4. 입력창에 해당 언어의 쿼리를 직접 입력하거나, 자연어 설명 후 **AI로 생성**으로 쿼리를 채웁니다
5. 쿼리를 검토한 뒤 **실행**(또는 입력창에서 **Enter**)을 눌러 조회합니다
6. 차트 또는 테이블로 나타난 결과를 확인합니다

:::info 데이터소스가 없을 때
연결된 데이터소스가 하나도 없으면 드롭다운이 비어 있고 **"설정된 데이터소스가 없습니다 — Datasources 탭에서 추가하세요."** 안내가 표시됩니다. 데이터소스 연결과 자격 증명 등록은 **연동(Integrations) 허브**(`/integrations` → 데이터소스 탭)에서 진행합니다.
:::

:::tip 빠른 실행
입력창에서 **Enter**를 누르면 바로 실행됩니다. 자연어 설명창에서 **Enter**를 누르면 **AI로 생성**이 실행됩니다.
:::

:::tip 결과가 잘렸다면
**결과가 잘렸습니다(상한 도달)** 경고가 보이면 쿼리 범위나 조건을 좁혀 다시 조회하세요.
:::

## AI 분석 팁
- "지난 5분간 모든 노드의 CPU 사용률" → PromQL 쿼리로 생성
- "error 를 포함하는 최근 로그" → LogQL 쿼리로 생성
- "500ms 이상 걸린 트레이스" → TraceQL 쿼리로 생성
- "테이블별 행 수" → 읽기 전용 SQL 로 생성

## 多语言支持 (i18n)

基于 React Context + localStorage，无需 URL 路由即可切换语言。该设计不会与 Next.js 的 `basePath: '/awsops'` 产生冲突。

<I18nArchitecture />

| 项目 | Before (v1.5) | After (v1.6.0) |
|------|---------------|-----------------|
| 语言 | 仅韩语 | 韩语 + 英语 |
| 翻译键 | 0 | **500+**（每种语言） |
| 语言切换 | 无 | Sidebar `EN/한` 切换按钮 |
| AI 响应语言 | 仅韩语 | 根据设置自动切换韩/英 |
| URL 变更 | - | 无（避免 basePath 冲突） |

:::info 为什么不使用基于 URL 的 i18n？
由于设置了 Next.js 的 `basePath: '/awsops'`，它会与 Next.js 内置 i18n 的 locale 前缀（`/ko/awsops`、`/en/awsops`）产生冲突。React Context + localStorage 方式不会改变 URL，从而规避了这一问题。
:::

**翻译范围**：
- 全部 35 个页面的文本
- Header、Sidebar 菜单标签
- AI 状态消息（分析中、Gateway 连接中、完成等）
- DataTable 列标题
- StatsCard 标签

## Bedrock 监控 (`/bedrock`)

通过 CloudWatch 指标和 AWSops 应用自身的 Token 跟踪，监控 Bedrock 各模型的用量。

<BedrockMetricsFlow />

| 功能 | 说明 |
|------|------|
| **各模型用量** | Claude Sonnet、Opus 等各模型的调用次数/Token |
| **Account vs AWSops** | AWSops 应用在 AWS 账户整体 Bedrock 用量中的占比 |
| **AI Token 成本** | 在聊天响应中显示输入/输出 Token 数 + USD 成本 |
| **提示词缓存** | 显示缓存命中率，确认成本节省效果 |
| **默认范围** | 7 天（可更改） |
| **API 路由** | `/api/bedrock-metrics` (`src/app/api/bedrock-metrics/route.ts`) |

:::tip AI Token 成本显示
在聊天响应底部以 `输入: 2,450 Token · 输出: 1,200 Token · $0.0234` 的形式显示成本。
用户可以实时确认每个问题的成本，从而提升成本意识。
:::

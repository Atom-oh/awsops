A major release that integrates external observability stacks as datasources, adds **natural-language-to-query** generation, and ships the **15-section AI comprehensive diagnosis** report.

<!--truncate-->

:::tip Highlights
**7 observability platforms** · **natural language → PromQL/LogQL/TraceQL/SQL** · **AI routing 10→11 (`datasource`)** · **15-section diagnosis + DOCX/PDF/schedule** · **SSRF protection**
:::

## External observability integration (7 platforms)

On top of AWS data, AWSops connects existing observability platforms as datasources.

| Type | Platform |
|------|----------|
| Metrics | Prometheus · Dynatrace · Datadog |
| Logs | Loki · ClickHouse |
| Traces | Tempo · Jaeger |

- `/datasources` — datasource CRUD · connection test · auth configuration
- `/datasources/explore` — direct query execution + **AI query generation (natural language → PromQL/LogQL/TraceQL/SQL)**
- The AI assistant's `datasource` route auto-classifies external-metric questions (AI routing expanded **10 → 11**)

## AI comprehensive diagnosis (15 sections)

- Bedrock Opus analyzes the whole infrastructure across **15 sections** — focused on cost/security/reliability
- Export to **DOCX · Markdown · browser Print-to-PDF**
- **Weekly/biweekly/monthly** auto-schedule (report scheduler)
- Print-friendly report page `/ai-diagnosis/report` (A4 page breaks)

## Security (SSRF protection)

- URL validation in `datasource-client.ts` protects all outbound fetch paths, including the AI route
- Allowlist-based private-network access control + IPv6 private/link-local detection (`fc00::/7`, `fe80::/10`)
- Datasource query and AI query generation are admin-only

## Also

- **Converse Stream API** for multi-route synthesis + typing effect for AgentCore responses
- Automatic zombie PostgreSQL connection cleanup; time-range filters on monitoring queries to prevent pg pool exhaustion
- EKS Access Entry status + kubeconfig registration, EKS service resources tab

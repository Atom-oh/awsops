# 登录指南

AWSops 仪表板使用基于 Amazon Cognito 的身份验证。您可以使用管理员提供的电子邮箱和密码进行登录。

## 登录流程

### 1. 访问仪表板

在浏览器中访问仪表板 URL：
```
https://your-domain.com/awsops/
```

### 2. 身份验证界面

当 CloudFront Lambda@Edge 检测到未经身份验证的请求时，会自动重定向到 Cognito 登录页面。

### 3. 输入凭证

- **电子邮箱**：管理员注册的电子邮箱地址
- **密码**：您设置的密码

:::tip 首次登录
首次登录时，需要将临时密码更改为新密码。密码必须满足以下条件：
- 至少 8 个字符
- 包含大写字母、小写字母、数字和特殊字符
:::

### 4. 完成登录

身份验证成功后，将跳转到仪表板主页面。

## 退出登录 (Sign Out)

退出登录通过侧边栏顶部的按钮完成。

### 退出登录位置
1. 找到屏幕左侧侧边栏顶部的 **AWSops** 徽标
2. 点击徽标右侧的**退出登录图标**（门形图标）

### 退出登录行为
- 点击后向服务器发送 `POST /api/auth` 请求
- HttpOnly Cookie 在服务器端被安全删除
- 自动重定向到登录页面

:::info HttpOnly Cookie
出于安全考虑，身份验证令牌存储在 HttpOnly Cookie 中。此 Cookie 无法通过 JavaScript 访问，从而防止 XSS 攻击。因此，退出登录时采用在服务器端删除 Cookie 的方式。
:::

## 会话管理

### 会话有效时间
- 默认会话有效时间遵循 Cognito 的配置
- 会话过期后会自动重定向到登录页面

### 多设备
- 同一账户可以在多台设备上同时登录
- 每台设备的会话独立管理

## 故障排查

### 无法登录时

| 症状 | 解决方法 |
|------|----------|
| 密码错误 | 请求重置密码 |
| 账户被锁定 | 联系管理员 |
| 页面无法加载 | 清除浏览器缓存后重试 |

### 退出登录后仍能访问时

1. 清除浏览器 Cookie
2. 在无痕/隐私模式下测试
3. 在其他浏览器中测试

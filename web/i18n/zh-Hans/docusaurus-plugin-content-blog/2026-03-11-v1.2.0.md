本次发布将 Gateway 从 7 个拆分为 **8 个 Gateway**，并大幅改进了部署稳定性。Container 响应速度**提升 54%**（50s → 23s），全部测试达成 **104/104 通过**。

:::tip 核心成果
**8 Gateway** · **10 Routes** · **104/104 测试通过** · **93% 分类准确率** · **Container 速度提升 54%**
:::

## Gateway 拆分：Infra → Network + Container

v1.1.0 的 Infra Gateway（41 个工具）包含的工具过多，因此将其拆分为网络和容器两个领域。

<GatewaySplitFlow />

| 项目 | Before（Infra 单一） | After（拆分后） |
|------|---------------------|-------------|
| Gateway | 1 (Infra, 41 个工具) | 2 (Network 17 + Container 24) |
| Container 响应时间 | ~50 秒 | ~23 秒（**提升 54%**） |
| 工具选择准确率 | LLM 从 41 个中选择 | 从 24 个中选择 |
| 路由体系 | 7 Gateway, 9 Routes | **8 Gateway, 10 Routes** |

:::info 为什么拆分更有效？
与让 LLM 阅读全部 41 个工具说明后再选择相比，从 17 个（Network）或 24 个（Container）中选择**更快也更准确**。特别是 Container 问题中误选网络工具的错误已经消失。
:::

## 部署稳定性改进

在部署脚本中添加了错误处理和自动检测逻辑。

| 脚本 | 改进内容 |
|----------|-----------|
| `06a-setup-agentcore-runtime.sh` | `run_or_fail` 辅助函数 + AWS 凭证预检（preflight check） |
| `06b-setup-agentcore-gateway.sh` | 批量创建 8 个 Gateway（反映 Network/Container 拆分） |
| `07-setup-cloudfront-auth.sh` | CloudFront 自动检测 — 优先 CDK 栈，回退到 ALB origin |
| Steampipe 连接 | 硬编码密码 → 从 `steampipe service status` 动态获取 |

## 主要 Bug 修复

| 问题 | 原因 | 修复 |
|------|------|------|
| EKS access entry 失败 | 使用了 `arn:aws:sts::` 格式 | 转换为 `arn:aws:iam::` 格式 |
| Cognito custom domain 错误 | 缺少 `SupportedIdentityProviders` | 添加该参数 |
| AgentCore 权限错误 | 权限不足 | 添加 `bedrock-agentcore:*` 权限 |
| K8s PVC 查询失败 | JSONB 序列化错误 | 在 `k8s.ts` 中应用 `::text` 类型转换 |

:::warning Steampipe JSONB 注意事项
Kubernetes 表的 JSONB 列（`spec`、`status`）如果直接返回会发生序列化错误。
必须使用 `SELECT spec::text` 或 `spec->>'key'` 进行转换。
:::
